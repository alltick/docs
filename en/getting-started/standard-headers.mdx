All AllTick APIs communicate using JSON. This page documents the common request and response envelope.

## REST API request format

GET requests pass a URL-encoded JSON payload via the `query` parameter:

```
GET /kline?token=YOUR_TOKEN&query=URL_ENCODED_JSON
```

JSON structure:

```json
{
  "trace": "unique-request-id",
  "data": {
    // Parameters vary by endpoint
  }
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| trace | string | Yes | Request correlation ID, returned in response. Max 64 characters |
| data | object | Yes | Request parameters, structure varies by endpoint |

## REST API response format

```json
{
  "ret": 200,
  "msg": "ok",
  "trace": "unique-request-id",
  "data": {}
}
```

| Field | Type | Description |
|-------|------|-------------|
| ret | integer | Status code. 200 = success |
| msg | string | Status message |
| trace | string | Matches the request trace |
| data | object | Response payload |

## WebSocket message format

```json
{
  "cmd_id": 22004,
  "seq_id": 1,
  "trace": "unique-request-id",
  "data": {}
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| cmd_id | integer | Yes | Protocol command ID |
| seq_id | integer | Yes | Sequence number, returned in response |
| trace | string | Yes | Request correlation ID |
| data | object | Yes | Request parameters |

## Error codes

| Code | Description | Resolution |
|------|-------------|------------|
| 200 | Success | — |
| 202 | Invalid request parameters | Check JSON format and fields |
| 401 | Invalid or expired token | Verify your token is correct |
| 429 | Rate limit exceeded | Reduce request frequency or upgrade plan |
| 600 | Product code not found | Check the symbol code |
| 601 | No permission for this product | Confirm your plan includes this symbol |

| Value | Interval |

</Note>
