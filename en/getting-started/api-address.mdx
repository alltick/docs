description: 'REST and WebSocket endpoint URLs by asset class'
---

AllTick separates endpoints by asset class. Stocks use a dedicated base URL; all other markets share a common one.

## REST API endpoints

| Asset class | Base URL |
|-------------|----------|
| US stocks, HK stocks, A-shares, indices | `https://quote.alltick.co/quote-stock-b-api` |
| Forex, crypto, precious metals, commodities | `https://quote.alltick.co/quote-b-api` |

Available paths:

| Path | Method | Description |
|------|--------|-------------|
| `/kline` | GET | Single product historical K-line query |
| `/batch-kline` | POST | Batch query latest 2 K-line bars |
| `/trade-tick` | GET | Latest trade price query |
| `/depth-tick` | GET | Latest order book depth query |

Example request:

```
https://quote.alltick.co/quote-stock-b-api/kline?token=YOUR_TOKEN&query=URL_ENCODED_JSON
```

## WebSocket endpoints

| Asset class | Connection URL |
|-------------|----------------|
| US stocks, HK stocks, A-shares, indices | `wss://quote.alltick.co/quote-stock-b-ws-api?token=YOUR_TOKEN` |
| Forex, crypto, precious metals, commodities | `wss://quote.alltick.co/quote-b-ws-api?token=YOUR_TOKEN` |

<Note>
When querying forex or crypto symbols, use `/quote-b-api` (REST) or `/quote-b-ws-api` (WebSocket). For stocks, use the paths containing `-stock-`. The request parameter structure is identical — only the base URL changes.
</Note>
