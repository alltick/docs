## <strong>1. IP に関する制限</strong>

#### 1.1 WebSocket の接続数は Token に許可された接続数に基づいて制限され、IP アドレス単位では制限されません。

* 例：ベーシックプランでは 1 つの Token につき WebSocket 接続は 1 本のみ許可されます。IP アドレス A からすでに WebSocket 接続を開始している場合、1. 同じ IP アドレス A から 2 本目の WebSocket 接続を開始しようとしても拒否されます。2. IP アドレス B から 2 本目の WebSocket 接続を開始しようとしても拒否されます。どちらも、ベーシックプランが 1 本の WebSocket 接続のみを許可しているためです。
* 例：アドバンスプランでは 1 つの Token につき WebSocket 接続が 3 本まで許可されます。IP アドレス A から 3 本を同時に開始しても、IP アドレス A、B、C から 1 本ずつ開始しても、合計接続数が 3 本を超えなければ利用できます。

#### 1.2 「株式・指数系データ」と「外国為替・貴金属・原油系データ」はリクエスト URL が異なります。2 種類のデータへ同時に接続しても 1 つの WebSocket 接続として計算され、同時にリクエストできます。

* 例：ベーシックプランでは 1 つの Token につき WebSocket 接続は 1 本のみですが、IP アドレス A から株式データの WebSocket 接続を開始済みの場合でも、同じ IP アドレス A または IP アドレス B から外国為替・貴金属系データの WebSocket 接続をもう 1 本開始できます。

## <strong>2. インターフェース呼び出し頻度制限</strong>

#### 2.1 各インターフェースの頻度制限

* <strong>最新約定価格（逐次 Tick）インターフェース：</strong>1 秒ごとに 1 回のみリクエストできます。
* <strong>板情報（Order Book）インターフェース：</strong>1 秒ごとに 1 回のみリクエストできます。

#### 2.2 同じ WebSocket 接続内で複数のインターフェースをリクエストする場合、送信間隔は<strong>少なくとも 1 秒</strong>必要です。

* 例：ユーザー A が 28 分 30 秒に WebSocket で「最新約定価格インターフェース、リクエストプロトコル番号 22004」のリクエストを送信し、同じ秒内に「最新板情報、リクエストプロトコル番号 22002」をもう一度送信しようとした場合、2 回目のリクエストはシステムに拒否されます。

#### 2.3 複数の WebSocket 接続で複数の WebSocket リクエストを開始する場合、各 WebSocket リクエストは<strong>少なくとも 3 秒</strong>間隔を空けてください。

* 例：ユーザー A がアドバンスプランを購入しており、同プランが 3 本の WebSocket 接続をサポートしている場合、28 分 30 秒に 1 本目の WebSocket を開始したら、3 秒以上間隔を空け、28 分 34 秒に 2 本目の WebSocket 購読を開始できます。2 本の WebSocket 購読が成功した後は、10 秒ごとに心拍を送信し続ければ、インターフェースはリアルタイムでデータをプッシュします。

#### 2.4 接続が切断され、繰り返し再接続が必要な場合：

* 無料プランユーザー：2 回の再接続試行の間隔は少なくとも 10 秒必要です。
* 有料ユーザー（ベーシック、アドバンス、プロフェッショナル、全香港株、全 A 株、全米国株を含む）：2 回の再接続試行の間隔は少なくとも 3 秒必要です。

## <strong>3. 接続数制限</strong>

* プランによって接続数の制限は異なります。詳細は以下の表のとおりです。
* 規定の制限を超えて接続を確立しようとした場合、超過分の接続試行は直接切断されます。

| プラン | WebSocket 接続数 |
| --- | --- |
| 無料 | WebSocket 接続は 1 本のみ |
| ベーシック | WebSocket 接続は 1 本のみ |
| アドバンス | WebSocket 接続を 3 本まで確立可能 |
| プロフェッショナル | WebSocket 接続を 10 本まで確立可能 |
| 全香港株 | WebSocket 接続を 10 本まで確立可能 |
| 全 A 株 | WebSocket 接続を 10 本まで確立可能 |
| 全米国株 | WebSocket 接続を 10 本まで確立可能 |

## <strong>4. 商品コード（code）購読制限</strong>

* 単一の WebSocket 接続で一度に購読できる商品コード（codes）の数には制限があります。詳細は以下の表のとおりです。
* 規定の購読上限を超えて購読しようとした場合、システムは制限数内の先頭リクエストデータのみを処理し、その他のデータを無視します。

| プラン | code 購読数制限 |
| --- | --- |
| 無料 | **最新約定価格（逐次 Tick）インターフェース：**最大 5 商品まで同時リクエスト可能<br />**板情報（Order Book）インターフェース：**最大 5 商品まで同時リクエスト可能 |
| ベーシック | **最新約定価格（逐次 Tick）インターフェース：**最大 100 商品まで同時リクエスト可能<br />**板情報（Order Book）インターフェース：**最大 100 商品まで同時リクエスト可能 |
| アドバンス | **最新約定価格（逐次 Tick）インターフェース：**最大 200 商品まで同時リクエスト可能<br />**板情報（Order Book）インターフェース：**最大 200 商品まで同時リクエスト可能 |
| プロフェッショナル | **最新約定価格（逐次 Tick）インターフェース：**最大 3000 商品まで同時リクエスト可能<br />**板情報（Order Book）インターフェース：**最大 3000 商品まで同時リクエスト可能 |
| 全香港株 | **最新約定価格（逐次 Tick）インターフェース：**最大 3000 商品まで同時リクエスト可能<br />**板情報（Order Book）インターフェース：**最大 3000 商品まで同時リクエスト可能 |
| 全 A 株 | **最新約定価格（逐次 Tick）インターフェース：**最大 3000 商品まで同時リクエスト可能<br />**板情報（Order Book）インターフェース：**最大 3000 商品まで同時リクエスト可能 |
| 全米国株 | **最新約定価格（逐次 Tick）インターフェース：**最大 3000 商品まで同時リクエスト可能<br />**板情報（Order Book）インターフェース：**最大 3000 商品まで同時リクエスト可能 |

#### 注意事項

* 上記の制限に従って WebSocket 接続とリクエスト戦略を計画し、不要なサービス中断を避けてください。
* これらの制限は、すべてのユーザーが公平かつ効率的にサービスへアクセスできるようにし、バックエンドサービスを不適切な負荷から保護するためのものです。
* 問題が発生した場合や追加サポートが必要な場合は、速やかに技術サポートチームへお問い合わせください。
