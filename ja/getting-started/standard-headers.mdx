AllTick のすべての API は JSON 形式で通信します。このページでは共通のリクエスト・レスポンスエンベロープを説明します。

## REST API リクエスト形式

GET リクエストは `query` パラメータで URL エンコードされた JSON を渡します：

```
GET /kline?token=YOUR_TOKEN&query=URL_ENCODED_JSON
```

JSON 構造：

```json
{
  "trace": "unique-request-id",
  "data": {
    // パラメータはエンドポイントにより異なる
  }
}
```

| フィールド | 型 | 必須 | 説明 |
|-----------|------|------|------|
| trace | string | はい | リクエスト追跡 ID、レスポンスで返却。最大 64 文字 |
| data | object | はい | リクエストパラメータ、構造はエンドポイントにより異なる |

## REST API レスポンス形式

```json
{
  "ret": 200,
  "msg": "ok",
  "trace": "unique-request-id",
  "data": {}
}
```

| フィールド | 型 | 説明 |
|-----------|------|------|
| ret | integer | ステータスコード。200 = 成功 |
| msg | string | ステータスメッセージ |
| trace | string | リクエストの trace と一致 |
| data | object | レスポンスペイロード |

## WebSocket メッセージ形式

```json
{
  "cmd_id": 22004,
  "seq_id": 1,
  "trace": "unique-request-id",
  "data": {}
}
```

| フィールド | 型 | 必須 | 説明 |
|-----------|------|------|------|
| cmd_id | integer | はい | プロトコルコマンド ID |
| seq_id | integer | はい | シーケンス番号、レスポンスで返却 |
| trace | string | はい | リクエスト追跡 ID |
| data | object | はい | リクエストパラメータ |

## エラーコード

| コード | 説明 | 対処方法 |
|--------|------|----------|
| 200 | 成功 | — |
| 202 | リクエストパラメータ無効 | JSON 形式とフィールドを確認 |
| 401 | Token 無効または期限切れ | Token が正しいか確認 |
| 429 | レート制限超過 | リクエスト頻度を下げるかプランをアップグレード |
| 600 | 製品コードが見つからない | 銘柄コードを確認 |
| 601 | この製品へのアクセス権限なし | プランに該当銘柄が含まれるか確認 |
