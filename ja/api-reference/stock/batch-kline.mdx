## POST /batch-kline

## **インターフェース説明**

このインターフェースは複数商品を一括照会でき、複数のKラインタイプ（1分、15分、30分など）も一括照会できます。**ただし、一括照会できるのは最新2本のKラインのみです。**

HTTPインターフェースでKラインを取得する場合は、`/kline` と `/batch-kline` を組み合わせて使用することを推奨します。

* まず、`/kline` インターフェースで履歴データをポーリング取得し、ローカルデータベースに保存します。
* その後、`/batch-kline` インターフェースで複数商品の最新2本Kラインを継続的に一括取得し、データベースを更新します。

この方法により最新Kラインをすばやく更新でき、履歴Kラインを頻繁にリクエストすることで頻度制限を受けることを避けられます。

## **商品の最新Kラインを一括照会する場合、一括照会パラメータが多いためbodyに配置します。URLパラメータにはtokenのみを保持します。**

## Bodyリクエストパラメータ

```json
{
  "trace": "c2a8a146-a647-4d6f-ac07-8c4805bf0b74",
  "data": {
    "data_list": [
      {
        "code": "700.HK",
        "kline_type": 1,
        "kline_timestamp_end": 0,
        "query_kline_num": 1,
        "adjust_type": 0
      },
      {
        "code": "GOOGL.US",
        "kline_type": 1,
        "kline_timestamp_end": 0,
        "query_kline_num": 1,
        "adjust_type": 0
      }
    ]
  }
}
```

## リクエストパラメータ

| 名称 | 位置 | 型 | 必須 | 説明 |
| --- | --- | --- | --- | --- |
| token | query | string | はい | tokenが不明な場合は、担当者へお問い合わせください。 |
| body | body | object | いいえ |  |
| » trace | body | string | はい | ログ照会に使用するトレースコードです。リクエストごとに一意にしてください。 |
| » data | body | object | はい |  |
| »» data_list | body | [object] | はい |  |
| »»» code | body | string | はい | codeリストを確認し、照会するcodeを選択してください: <a href="https://docs.google.com/spreadsheets/d/1avkeR1heZSj6gXIkDeBt8X3nv4EzJetw4yFuKjSDYtA/edit?gid=495387863#gid=495387863">[codeリストを開く]</a><br />**注意: code値の大文字・小文字は商品リスト内のcodeと一致させる必要があります。** |
| »»» kline_type | body | integer | はい | Kラインタイプ:<br />1. 1は1分K、2は5分K、3は15分K、4は30分K、5は時間足、6は2時間K（株式は2時間K非対応）、7は4時間K（株式は4時間K非対応）、8は日足、9は週足、10は月足です。注: 株式は2時間K、4時間Kに対応していません。<br />2. 最短のKラインは1分のみ対応しています。<br />3. **前日終値**を照会する場合、kline_type に8を渡します。 |
| »»» kline_timestamp_end | body | integer | はい | 指定時刻から過去に向かってKラインを照会します。<br />1. 0を渡すと最新取引日から過去に向かって照会します。<br />2. タイムスタンプを渡すと、そのタイムスタンプから過去に向かって照会します。<br />3. タイムスタンプ指定に対応しているのは外国為替、貴金属、暗号資産のみで、株式系codeは対応していません。 |
| »»» query_kline_num | body | integer | はい | 1. 照会するKライン本数を示します。**このインターフェースでは最大2本のKラインのみ照会できます。**<br />2. このフィールドで**前日終値**を照会できます。kline_type に8、query_kline_num に2を渡すと、返却された2本のKラインのうちタイムスタンプが小さいデータが前日終値です。 |
| »»» adjust_type | body | integer | はい | 復権タイプです。株式系codeの場合のみ有効です。例: 0: 除権、1: 前方復権。現在は0のみ対応しています。 |
