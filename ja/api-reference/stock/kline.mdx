## GET /kline

## **インターフェース説明**

このインターフェースは履歴Kラインの照会に使用できます。ただし、1回のリクエストで照会できる商品は1つのみです。**取得した履歴Kラインはローカルデータベースにキャッシュすることを推奨します。**

HTTPインターフェースでKラインを取得する場合は、`/kline` と `/batch-kline` の2つのインターフェースを組み合わせて使用することを推奨します。手順は次のとおりです。

* まず、`/kline` インターフェースで履歴データをポーリング取得し、ローカルデータベースに保存します。以後の履歴データは顧客側データベースから直接取得でき、再度インターフェースを呼び出す必要はありません。
* その後、`/batch-kline` インターフェースを継続的に使用し、複数商品の最新2本のKラインを一括取得してデータベースを更新します。

この方法により最新Kラインをすばやく更新でき、履歴Kラインを頻繁にリクエストすることで頻度制限を受けることも避けられます。

## queryリクエストパラメータ

```json
{
  "trace": "3baaa938-f92c-4a74-a228-fd49d5e2f8bc-1678419657806",
  "data": {
    "code": "857.HK",
    "kline_type": 1,
    "kline_timestamp_end": 0,
    "query_kline_num": 2,
    "adjust_type": 0
  }
}
```

| 名称 | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| trace | string | はい | ログ照会に使用するトレースコードです。リクエストごとに一意にしてください。 |
| data | object | はい |  |
| » code | string | はい | codeリストを確認し、照会するcodeを選択してください: <a href="https://docs.google.com/spreadsheets/d/1avkeR1heZSj6gXIkDeBt8X3nv4EzJetw4yFuKjSDYtA/edit?gid=495387863#gid=495387863">[codeリストを開く]</a><br />**注意: code値の大文字・小文字は商品リスト内のcodeと一致させる必要があります。** |
| » kline_type | integer | はい | Kラインタイプ:<br />1. 1は1分K、2は5分K、3は15分K、4は30分K、5は時間足、6は2時間K（株式は2時間K非対応）、7は4時間K（株式は4時間K非対応）、8は日足、9は週足、10は月足です。注: 株式は2時間K、4時間Kに対応していません。<br />2. 最短のKラインは1分のみ対応しています。<br />3. **前日終値**を照会する場合、kline_type に8を渡します。 |
| » kline_timestamp_end | integer | はい | 指定時刻から過去に向かってKラインを照会します。<br />1. 0を渡すと、現在最新の取引日から過去に向かってKラインを照会します。<br />2. 指定時刻を使う場合はタイムスタンプを渡し、そのタイムスタンプから過去に向かって照会します。<br />3. タイムスタンプ指定に対応しているのは外国為替、貴金属、暗号資産のみで、株式系codeは対応していません。 |
| » query_kline_num | integer | はい | 1. 照会するKライン本数を示します。1回のリクエストで最大500本まで取得でき、タイムスタンプを使って過去方向に繰り返し取得できます。<br />2. このフィールドで**前日終値**を照会できます。kline_type に8、query_kline_num に2を渡すと、返却された2本のKラインのうちタイムスタンプが小さいデータが前日終値です。 |
| » adjust_type | integer | はい | 復権タイプです。株式系codeの場合のみ有効です。例: 0: 除権、1: 前方復権。現在は0のみ対応しています。 |
