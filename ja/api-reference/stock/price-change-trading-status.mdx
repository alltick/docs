## 騰落率説明

AllTickインターフェースは、騰落率または24時間騰落率フィールドを提供していません。お客様はAllTickデータを取得して騰落率を自行計算できます。

1. **日次騰落率の計算方法**:

   * **方法1**: HTTPインターフェースで当日の日足Kライン終値と前日の終値を取得します。計算式は次のとおりです。

   騰落率 = （当日終値 - 前日終値） / 前日終値 * 100%

   * **方法2**: WebSocketインターフェースで最新価格を取得し、HTTPインターフェースで前日の日足Kライン終値を取得します。計算式は次のとおりです。

   騰落率 = （最新価格 - 前日終値） / 前日終値 * 100%

2. **24時間騰落率の計算方法**:

   * WebSocketの最新約定価格インターフェース（リクエストプロトコル番号: 22004）を使用して、Tick単位の約定価格をリアルタイムで受信します。
   * 以後の計算に使用するため、WebSocketインターフェースからプッシュされた24時間前の最新価格を保存しておく必要があります。
   * 計算式:

   24時間騰落率 = （最新価格 - 24時間前の最新価格） / 24時間前の最新価格 * 100%

## 株式ストップ高・ストップ安説明

AllTickはストップ高またはストップ安を判定するインターフェースを提供していません。お客様は次の方法で株式がストップ高またはストップ安かどうかを判定できます。

1. **板情報インターフェースデータによる判定**: 板情報インターフェースを購読している場合、片側のみにデータがあり、もう一方の価格と数量がすべて0を返すとき、ストップ高またはストップ安と判定できます。具体的な判定方法は次のとおりです。
   * `bid` 側のみデータがあり、すべての `ask` 価格と数量が0を返す場合、その株式はストップ高です。
   * `ask` 側のみデータがあり、すべての `bid` 価格と数量が0を返す場合、その株式はストップ安です。

## 株式上場廃止判定説明

AllTickは上場廃止判定インターフェースを提供していません。お客様は次の方法で株式が上場廃止かどうかを判定できます。

**判定方法**: 板情報インターフェースを購読している場合、`bid` と `ask` の両側の価格と数量がすべて0を返すと、その株式は上場廃止済みであることを示します。
