---
title: '请求格式'
description: 'REST 和 WebSocket 的请求与响应格式规范'
---

AllTick 所有接口使用 JSON 格式通信。本页说明通用的请求和响应结构。

## REST API 请求格式

GET 请求通过 `query` 参数传递 URL 编码后的 JSON：

```
GET /kline?token=YOUR_TOKEN&query=URL_ENCODED_JSON
```

JSON 结构：

```json
{
  "trace": "unique-request-id",
  "data": {
    // 具体参数见各接口文档
  }
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| trace | string | 是 | 请求追溯 ID，响应原样返回，最长 64 字符 |
| data | object | 是 | 请求参数，结构因接口而异 |

## REST API 响应格式

```json
{
  "ret": 200,
  "msg": "ok",
  "trace": "unique-request-id",
  "data": {}
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| ret | integer | 状态码，200 表示成功 |
| msg | string | 状态描述 |
| trace | string | 与请求中的 trace 一致 |
| data | object | 响应数据 |

## WebSocket 消息格式

```json
{
  "cmd_id": 22004,
  "seq_id": 1,
  "trace": "unique-request-id",
  "data": {}
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| cmd_id | integer | 是 | 协议号，标识操作类型 |
| seq_id | integer | 是 | 序列号，响应原样返回 |
| trace | string | 是 | 请求追溯 ID |
| data | object | 是 | 请求参数 |

## 错误码

| 错误码 | 说明 | 处理方式 |
|--------|------|----------|
| 200 | 成功 | — |
| 400 | 请求 JSON 第一层参数或 data 字段参数无效 | 检查 JSON 结构、必填字段和 data 对象 |
| 401 | Token 无效 | 检查 Token 格式、账号状态和是否过期 |
| 402 | query 参数无效 | 检查 GET 请求的 query 参数并确认已 URL 编码 |
| 429 | 超出频率限制 | 降低请求频率或升级套餐 |
| 600 | 产品代码无效 | 检查请求 URL 与 code 是否匹配，并核对产品列表 |
| 601 | 请求消息体为空 | 检查 POST 请求 body，特别是 `/batch-kline` |
| 603 | Token 等级不足 | 检查请求产品数、K 线根数或套餐限制 |
| 604 | 无权限访问该产品 | 确认套餐是否包含该 code |
| 605 | 请求频率限制 | 降低请求频率或升级套餐 |
| 606 | 请求过频且连接将关闭 | WebSocket 请求间隔至少 3 秒，并检查连接数限制 |

## K 线类型枚举

| 值 | 周期 |
|----|------|
| 1 | 1 分钟 |
| 2 | 5 分钟 |
| 3 | 15 分钟 |
| 4 | 30 分钟 |
| 5 | 1 小时 |
| 6 | 2 小时 |
| 7 | 4 小时 |
| 8 | 日 |
| 9 | 周 |
| 10 | 月 |

<Note>
K 线类型 6（2 小时）和 7（4 小时）不适用于股票类产品。
</Note>
