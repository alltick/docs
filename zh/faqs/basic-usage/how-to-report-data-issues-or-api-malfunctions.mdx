报告问题之前，请尽可能收集和整理详细的问题信息，包括：

* 出现问题的具体时间和日期。
* 受影响的API端点。
* 发送的请求示例，并移除API密钥等敏感信息。
* 接收到的响应，包括错误代码和消息。
* 问题重现步骤。

建议先查看官方文档、FAQ或开发者社区，确认是否已有解决方案。若仍需报告问题，请通过官网联系表单、客户支持邮箱或支持票务系统提交，并提供上述信息以帮助技术团队快速定位和解决问题。

报告问题时保持沟通专业、清晰和完整，可以加快问题解决速度，也有助于改进产品和服务。
